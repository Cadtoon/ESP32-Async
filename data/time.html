Time, too much, too little, too soon, too late, it's about time.

The thought of putting in time was in a way to demonstrate the use of different inputs, but it falls short of doing that, but not really. A tutorial I followed was demonstrating how to use strings, integers, and floats. Well, from everything I have done so far, the only thing you can pass between an App and a webpage is text. If you want to use it in a different format you must convert it once it has been passed over. The alarm time, it's converted to text before it is sent here. The time display at the top of the page, it's just text with a fancy font. When you select a time from the input and click the update, it is sent as text. The alarm on-off, you will notice in the grey area that it is stored as a 0 or 1. It is much easier to convert a 0 or 1 to a bool then on or off, true or false. Much the same applies to saving the variables in a text file. if it is anything other than text, it must be converted before you can use it, and in some cases, it needs to be converted to a const string.

The alarm time and alarm on/off have no functionality in this App. The NTP server and zone, they don't really serve any functionality in App either, other than to display the ESPs time and data on these webpages. However, it become very useful if your App needs any kind of scheduling or is a seven-segment clock.

I have added links that contain list for both time servers and time zones. I have found this to be the most efficient way of setting the time. Other methods require the number of minutes offset from GMT to calculate the time, and the number of minutes to offset for DST. These are numbers you would probably need to Google to find anyway.
